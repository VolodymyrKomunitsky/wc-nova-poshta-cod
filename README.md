# WC Nova Poshta COD

> Метод оплати **«Накладений платіж (Нова Пошта)»** для WooCommerce.  
> Плагін розширює WooCommerce через офіційні хуки та фільтри — ядро WordPress/WooCommerce **не змінюється**.

---

## Можливості

| # | Функція |
|---|---------|
| 1 | Власний метод оплати (`wcnpc_cod`) із відображенням у WooCommerce → Payments |
| 2 | Сторінка налаштувань: назва, опис, інструкції, максимальна сума, тип і розмір комісії |
| 3 | Власні поля на чекауті — місто та номер відділення НП |
| 4 | **Серверна валідація** полів (PHP) + **клієнтська** inline-валідація (JS) |
| 5 | Збереження міста та відділення в метаданих замовлення (HPOS-сумісно) |
| 6 | Відображення даних НП в адмін-картці замовлення та листах покупцю |
| 7 | Сумісність із High-Performance Order Storage (HPOS) |
| 8 | Перевірка активності WooCommerce — попередження замість фатальної помилки |

---

## Вимоги

- PHP **8.0** або новіший
- WordPress **6.0** або новіший
- WooCommerce **7.0** або новіший

## Встановлення

### Варіант A — завантаження ZIP (рекомендовано для продакшену)

1. Завантажте останній реліз: `wc-nova-poshta-cod.zip`
2. В адмін-панелі WordPress перейдіть до  
   **Плагіни → Додати новий → Завантажити плагін**
3. Оберіть завантажений ZIP і натисніть **Встановити зараз**
4. Натисніть **Активувати плагін**

### Варіант B — через FTP / файловий менеджер

```
wp-content/
└── plugins/
    └── wc-nova-poshta-cod/       ← папка плагіну
        ├── wc-nova-poshta-cod.php
        ├── includes/
        ├── assets/
        └── languages/
```

1. Скопіюйте папку `wc-nova-poshta-cod` у `/wp-content/plugins/`
2. Активуйте в **Плагіни → Встановлені плагіни**

### Варіант C — WP-CLI

```bash
wp plugin install ./wc-nova-poshta-cod.zip --activate
```

## Налаштування

1. Перейдіть до **WooCommerce → Settings → Payments**
2. Знайдіть **«Накладений платіж (Нова Пошта)»** і натисніть **Manage**
3. Налаштуйте параметри:

| Поле | Опис | За замовч. |
|------|------|-----------|
| Увімкнено | Активувати метод | ✔ |
| Назва | Текст для покупця на чекауті | Накладений платіж (Нова Пошта) |
| Опис | Короткий опис під назвою | — |
| Інструкції | Текст на сторінці подяки та у email | — |
| Макс. сума (грн) | 0 = без обмежень | 50 000 |
| Тип комісії | Без / Фіксована / Відсоток | Без комісії |
| Розмір комісії | Значення залежно від типу | 0 |

4. Натисніть **Save changes**

## Дані, що зберігаються в замовленні

| Мета-ключ | Вміст |
|-----------|-------|
| `_wcnpc_np_city` | Місто доставки |
| `_wcnpc_np_branch` | Номер відділення |

Дані зберігаються через `WC_Order::update_meta_data()` — сумісно з **HPOS**.

## Хуки та фільтри (для розробників)

Плагін **не патчить** і **не перевизначає** функції ядра. Використовуються виключно:

```php
// Реєстрація методу оплати
add_filter( 'woocommerce_payment_gateways', 'wcnpc_register_gateway' );

// Збереження налаштувань
add_action( 'woocommerce_update_options_payment_gateways_wcnpc_cod', ... );

// Відображення у замовленнях та листах
add_action( 'woocommerce_admin_order_data_after_billing_address', ... );
add_action( 'woocommerce_email_after_order_table', ... );
add_action( 'woocommerce_email_before_order_table', ... );
add_action( 'woocommerce_thankyou_wcnpc_cod', ... );
```

## Деактивація та видалення

- **Деактивація** — метод оплати зникає з чекауту; дані існуючих замовлень зберігаються.  
- **Видалення** — видаліть плагін через адмін-панель. Метадані замовлень (`_wcnpc_np_*`) залишаються в БД (не видаляються навмисно, щоб зберегти історію).
